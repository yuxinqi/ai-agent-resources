
## 报告格式

每份基准测试报告包含以下部分：

| 部分 | 说明 |
|------|------|
| **测试概览** | 测试范围、日期、使用的模型和工具 |
| **测试配置** | 模型版本、参数设置、测试环境 |
| **结果摘要** | 各维度的评分和整体评级 |
| **详细结果** | 每个测试用例的具体结果 |
| **趋势分析** | 与历史结果的对比 |
| **建议** | 基于结果的改进建议 |

## 评级体系

基准测试报告使用统一的评级体系：

| 评级 | 分数范围 | 说明 |
|------|---------|------|
| A | 90-100 | 优秀，生产环境可用 |
| B | 70-89 | 良好，主要场景可用 |
| C | 50-69 | 一般，需改进后使用 |
| D | 0-49 | 较差，不建议使用 |

## 报告归档规则

1. 报告按月归档至 `YYYY-MM/` 目录
2. 文件名格式：`{test-type}-summary-{YYYYMM}.md`
3. 同一测试类型每月生成一份汇总报告
4. 重要变更可生成临时报告

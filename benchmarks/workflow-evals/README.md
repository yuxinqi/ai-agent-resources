# Workflow Evals — 工作流评估基准

> 本目录存放工作流（Workflow）的端到端评估测试定义和结果。

## 评估维度

| 维度 | 说明 | 评估方法 |
|------|------|---------|
| **任务完成率** | 工作流是否能完整完成目标任务 | 端到端验证 |
| **步骤效率** | 完成任务所需的最少步骤 vs 实际步骤 | 步骤数统计 |
| **输出质量** | 最终输出的准确性和完整性 | 人工评分 + LLM-as-Judge |
| **鲁棒性** | 对异常情况和输入变化的处理能力 | 异常注入测试 |
| **成本效率** | Token 消耗和 API 调用次数 | 自动统计 |
| **可恢复性** | 失败后是否能从断点恢复 | 中断恢复测试 |

## 评估场景格式

```yaml
name: "research-agent-eval"
workflow: "Research Agent"
description: "研究代理工作流端到端评估"
scenarios:
  - name: "simple_research"
    task: "对比 GPT-4o 和 Claude 4 Sonnet 的定价"
    complexity: "low"
    max_steps: 5
    max_tokens: 15000
    pass_criteria:
      - "包含两者的定价信息"
      - "信息来源可追溯"
      - "5 步内完成"
  - name: "complex_research"
    task: "设计一个 RAG 系统架构，含技术选型和可行性分析"
    complexity: "high"
    max_steps: 12
    max_tokens: 50000
    pass_criteria:
      - "覆盖需求分析、技术选型、架构设计"
      - "包含可行性评估"
      - "12 步内完成"
```

## 评级标准

| 评级 | 说明 | 条件 |
|------|------|------|
| A | 工作流高效完成，输出优质 | 完成率≥90% + 效率≥80% + 质量≥85% |
| B | 工作流完成，偶有冗余步骤 | 完成率≥75% + 效率≥60% + 质量≥70% |
| C | 工作流基本完成，效率或质量有不足 | 完成率≥50% + 效率≥40% + 质量≥50% |
| D | 工作流无法完整完成或输出质量差 | 不满足 C 级条件 |

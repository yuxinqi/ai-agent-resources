## 指标结果

| 指标 | 数值 | 说明 |
|------|------|------|
| 常见问题检出率 | 70% (GPT-4o) / 80% (Claude) | 5 个预埋问题中的检出比例 |

| 建议可操作性 | 78% | 建议可直接应用的比例 |

## 成功情况

1. 对常见编码问题（空指针、资源泄漏、SQL 注入）识别能力较强
2. Claude 在安全漏洞检测方面表现优于 GPT-4o
3. 风格建议整体合理，大部分可直接采纳
4. 输出格式清晰，便于开发者快速定位问题

## 失败情况

1. 竞态条件等并发问题识别能力弱
2. 复杂业务逻辑 Bug 基本无法识别
3. GPT-4o 对 CSRF 类安全问题敏感度不足
4. 对大型代码文件（>500 行）的审查质量下降明显

## 成本与延迟

| 指标 | 数值 |
|------|------|
| 单次审查平均延迟 | GPT-4o: 4.2s / Claude: 5.1s |
| 单次审查 Token 消耗 | ~3,000-8,000 tokens（视代码量） |
| 100 行代码审查估算成本 | ~$0.03-0.08 |

## 风险与限制

1. AI 代码审查不能替代人工审查，仅作为辅助工具
2. 安全漏洞检测存在漏报风险，关键系统必须配合专业安全扫描
3. 大文件审查效果下降，建议分段审查
4. 模型可能对某些框架/语言的审查能力不均衡

## 复现步骤

1. 准备测试代码集（5 段含已知问题的 Python/TypeScript 代码）
2. 使用代码审查提示词模板
3. 分别用 GPT-4o 和 Claude 4 Sonnet 进行审查
4. 比对输出与预埋问题清单
5. 统计检出率、误报率等指标
